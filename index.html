Group recommender system supporting music selection

A system that provides personalised recommendation for a group of users.

The following technologies were used: MongoDB, Spotify API, Python (SciPy, NumPy, Pandas, Matplotlib, PyMongo, Flask)

Data Preparation – the challenges:
  • Representing users and tracks – users and track were represented as data point in the same 7-dimensional space.
  • Determining users’ music taste – calculated from the weighted average of their most popular songs' features.
  • Grouping users by their music taste – K-means clustering was used to group the users.
  • Getting the most representative users of each group - Euclidean distance was used to designate a leader of each group - the closest user to the centroid.
Making Recommendations – the challenges:
  • Comparing tracks with users’ taste – dot product of tracks features and groups’ music taste was used
  • Determining the groups’ music taste – calculated by adding the taste of all users in the group with their group leaders. The sum is then divided by two times the number of members in the group.

To see a more descriptive explanation, go to the GitHub repository and see the README.md file.

Hot Wheels Value Prediction System (Work in Progress)

End-to-end project combining data engineering and data science for collectible cars.

This personal project focuses on building a system that scrapes, processes, analyses, and eventually predicts future value of diecast model cars (e.g. Hot Wheels 1/64 scale). It uses real-world data from hobbyDB, a crowdsourced collectible database.

Work in progress – currently includes:
  • Data collection – using Scrapy spiders and public API endpoints to extract product metadata.
  • JSON processing – nested field access with safe navigation (.get(), next()) to handle inconsistencies.
  • Feature extraction – identifying key value drivers like Rubber Tyres, Limited Editions, Color variants, etc.
  • Value prediction (coming soon) – regression models to estimate future value based on features and historical price trends.
  • Automation (planned) – building a full data pipeline (using Airflow or Prefect) to regularly update the data.
  • Cloud deployment (planned) – hosting the system on AWS for scalability and automation.

This is an exploratory playground where I combine my skills in Python, data pipelines, cloud, web scraping, and machine learning.

Project not yet public – code and predictions will be shared once a minimum viable version is complete.
